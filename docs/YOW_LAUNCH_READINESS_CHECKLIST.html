1. Exact Next Task Queue

Do these in this order. Do not skip ahead to feature building until Task 4 is complete.

Task 1: Get a safe local test environment working.

  • Start YOW in offline/dev auth mode or use a disposable test account.
  • Open the app dashboard, not just the public homepage.
  • If the app cannot reach the dashboard, fix that before doing anything else.
Done when: you can create projects without using a real customer account.

Task 2: Run the 10 project type create/open/export smoke test.

  • Create one project for each type: Novel, Novella, Short Story, D&D Campaign, TTRPG Campaign, Play, Screenplay, TV Series, Comic / Graphic Novel, Video Game.
  • For each one: open it, type one unique sentence, refresh, confirm the sentence remains, export ZIP, export DOCX, export PDF if offered.
  • Write down every failure immediately.
Done when: all 10 types either pass or each failure has a specific bug note.

Task 3: Fix only smoke-test failures.

  • If a project type cannot be created, opened, refreshed, or exported, fix that exact issue.
  • If a label is misleading, fix the label.
  • If export crashes, fix export before building new features.
  • After each fix, rerun only the failed smoke test.
Done when: Task 2 passes for all 10 project types.

Task 4: Run the beta safety regression.

  • Test autosave: type in manuscript, refresh immediately, confirm text remains.
  • Test worldbuilding saves: create one character, location, lore entry, timeline event, and world history entry; refresh after each.
  • Test restore: export a ZIP, restore it, compare the restored project against the original.
  • Test responsive layout at desktop, tablet, and phone widths.
  • Test sign up, sign in, sign out, invalid login, and session refresh with a test account.
Done when: no beta blocker remains in data, auth, save, export, editor, responsive layout, or legal/payment basics.

Task 5: Build the final script engine.

  • Finish Play, Screenplay, and TV script editing controls.
  • Make script defaults feel correct immediately on new projects.
  • Make DOCX script output polished and readable.
  • Decide whether final Screenplay export needs FDX, screenplay PDF, or both.
  • Remove beta limitation copy only when script workflows are truly final.
Done when: Play, Screenplay, and TV script drafting/export feel like real supported workflows, not renamed prose scenes.

Task 6: Build the TV Series episode engine.

  • Add episode metadata: season, episode number, title, logline, status, runtime, POV/focus, draft notes.
  • Add a season/episode planning view.
  • Add series-bible fields that do not conflict with the existing book Series feature.
  • Make TV exports include episode metadata in a readable order.
Done when: a TV Series project feels like planning a show, not a novel with different labels.

Task 7: Build the D&D/TTRPG session engine.

  • Add structured session planning fields: hook, summary, encounters, NPCs, locations, secrets, rewards, consequences.
  • Add session recap fields.
  • Make D&D language fantasy/D&D-flavoured and TTRPG language system-neutral.
  • Make campaign exports include session plans and recaps.
Done when: campaign projects support actual session preparation and recap, not only loose notes.

Task 8: Build the Comic / Graphic Novel panel engine.

  • Add page records with page number, summary, setting, cast, and art notes.
  • Add panel records under each page.
  • Each panel needs panel number, shot/framing, action, caption, dialogue, SFX, and art direction.
  • Make exports preserve page and panel order.
Done when: a comic project supports page/panel planning as a first-class workflow.

Task 9: Build the Video Game narrative engine.

  • First decide the exact final scope: narrative bible only, quests/missions, dialogue trees, branching paths, or some combination.
  • Start with narrative bible fields: premise, world rules, factions, player role, core loop, locations, characters, quests/missions.
  • If branching is included, build a small scoped branch model before making a big visual graph.
  • Make exports include game narrative structure clearly.
Done when: a Video Game project supports the actual game narrative planning scope you chose.

Task 10: Run final full-site QA.

  • Rerun the 10 project type smoke test.
  • Rerun autosave, restore, export, auth, responsive, and large-project tests.
  • Audit homepage/pricing/legal copy and remove beta language only where features are complete.
  • Run npm run build.
Done when: the site is safe, fully specced, honestly described, and production build passes.

3. Final Feature Waves

After Tasks 1-4 are green, build these waves in order.

Wave 1: Finish script engine.

  • Finish Play-specific stage directions and dialogue behavior.
  • Finish Screenplay formatting and export decision.
  • Finish TV script drafting basics shared with the episode engine.
  • QA keyboard shortcuts, Enter flow, mobile layout, and DOCX output.

Wave 2: Build TV Series episode engine.

  • Build episode metadata.
  • Build season/episode overview.
  • Build series-bible fields.
  • Update exports.

Wave 3: Build tabletop session engine.

  • Build session plan fields.
  • Build recap fields.
  • Connect NPCs, locations, factions, rewards, and consequences.
  • Update exports.

Wave 4: Build comic panel engine.

  • Build page planning.
  • Build panel planning.
  • Add captions, dialogue, SFX, and art direction.
  • Update exports.

Wave 5: Build video game narrative engine.

  • Choose final scope.
  • Build narrative bible fields.
  • Add quests/missions/dialogue/branching only if they are in final scope.
  • Update exports.

Wave 6: Final polish and launch candidate.

  • Remove beta labels only for complete workflows.
  • Run full 10-type smoke again.
  • Run auth, save, export, restore, responsive, legal/payment QA again.
  • Run npm run build.

6. Recommended Implementation Order

If you want to reach a fully specced final launch efficiently, follow this order. First prove safety, then build the larger engines.

Finish the 10 project type UI smoke test.
Reason: all project types are selectable, so every type must create, open, survive refresh, and export without crashing.
Run autosave and data safety regression.
Reason: data loss is the scariest launch blocker. Test manuscript plus characters, locations, lore, timeline, world history.
Run export and restore regression.
Reason: ZIP restore is your backup story. It must work before launch.
Run auth and membership regression.
Reason: public users must be able to enter, leave, return, pay, and understand limits.
Run responsive pass.
Reason: launch copy says works on any device. Core flows must work at phone/tablet/desktop widths.
Fix blockers found during QA.
Reason: this gives you a safe beta foundation.
Implement the final-launch engines in waves.
Reason: script, TV, tabletop, comic, and game systems should be built deliberately, not all tangled together.
Run full final-launch QA after each wave.
Reason: every new engine must preserve data safety, exports, responsive layout, and honest copy.
Do one final copy audit.
Reason: once engines are complete, remove beta limitations only where the feature is genuinely finished.
Run production build and a final smoke.
Reason: final launch candidate should build cleanly and open without obvious runtime errors.
